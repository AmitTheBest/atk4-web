Web User Interface

Graphical User Interfaces have been around for decades. The idea is simple — implement a unified set of visual tools allowing application to communicate with User in expected way. Those visual tools are called "Views" in Agile Toolkit: Buttons, Frames, Menus, Tabs and others. Yet many other Web Frameworks failed to implement a consistent User Interface.

Why?

One reason is that some attempts to create Web User Interface but forget that they are making "Web" interface. Many desktop patterns do not apply. Windows and dragging is not always necessary. Simplicity and ability to customize is highly valued. Customization and unique look is required in almost every application. Another reason is PHP developer average skill-set. A typical PHP developer is coming from Design background and would know very little about programming. By the time they learn to program, they fail to find a decent framework written in PHP and they need to migrate to another language.

Agile Toolkit is for developers who LOVE their code. If you value performance, elegance and simplicity of your code, then you will feel at-home with Agile Toolkit. Your ability to recognize certain objects as "well made" and then using them to build higher-level layer of functionality is the great way to develop software and is very essential to your carrier. Agile Toolkit will teach you how.

SMLite - Agile's approach to Template Objects

Template is a common practice to separate HTML markup from business logic. Unfortunately popular template engines such as Smarty have decided to implement a new programming language inside a template. Agile Toolkit Templates reverse this pattern and make templates incredibly simple. Consider the following example:

Templates are <?tag?>simple<?/?>

The above string is a valid template in Agile Toolkit. You can manipulate tag values by manipulating them.

echo $tpl->set('tag','powerful')->render(); // outputs "Templates are powerful"

This is as simple as templates can get. It is also all we need from the template engine — ability to get and set region values. This simplicity is significant to the way how templates are used by the User Interface.

View - your Window into HTML Widget

$view = $page->add('Alex');

The web page you see in front of you consists of Views. View are implemented as objects and each view display a rectangular block on the web page. Above there was a Menu object. Below are some Form objects but to the right of this paragraph you will see a custom-made View implemented as a class "Alex".

Every object is capable of producing a chunk of HTML code which fits inside it's parent. Objects are rendered recursively by Agile Toolkit. Most of the objects rely on SMLite template to get the actual HTML code they need to output.

$view = $page->add('Alex');

The code above is the fundamental building block of your application in Agile Toolkit. You don't work with HTML, you work with objects. Agile Toolkit is bundled with many cool objects and in every application you are building, you'll be adding more of your own Views. Often views rely on custom SMlite template and incorporate other objects into that template. Together you get a consistent, unified and simple User Interface.
