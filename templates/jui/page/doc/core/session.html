Session Management

Agile Toolkit relies on standard PHP sessions, however it adds a object-oriented interface allowing objects to have local variables.

$o1 = $this->add('Button'); $o2 = $this->add('Button'); $o1->memorize('test',123); $o2->memorize('test',555); echo $o1->recall('test'); // outputs 123

Non-conflicting unique session storage is essential when you are building state-full objects. A good example is the "hangman" game demonstrated on the index page of agiletoolkit.org. Since the game is implemented as a view, it stores "guesess" locally and will not conflict with any other instances of the same class.
