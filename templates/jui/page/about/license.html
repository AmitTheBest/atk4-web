Dual Licensing of Agile Toolkit

Agile Toolkit offers licenses for commercial and open source AGPL development. license, commercial, agpl, open source, start ups, enterprises, freelancers

Agile Toolkit offers free and commercial licenses. Commercial license is required only if you want other to use teh product but not being able to acces source code.

When to use Open-source License?

Our Affero General Public License is suitable for open-source and many types of commercial products too:

  • Build some software for yourself. You do not need to disclose code.
  • Build back-end for your company, for internal use.
  • Create your own Open-Source CRM and use public github repository. Use site commecrially.
  • Build a frontend for your SaaS or PaaS project. Publish source on public github repository. Charge users subscription.

Technically - our AGPL license does not prevent you from commercial use. It requires however, that users of your software should be able to get the source code.

Code Ownership

Any code you develop using Agile Toolkit belongs to you. You may at any point remove your source codes from github and if you want to continue offering your service online, switch to closed-source license.

How to disclose code?

Create a github.com repository and add link to it somewhere in the "about" section of your project. Remember that if someone is using your code an building upon it, he is still bound to publish the code which you can then use. What if I don't like Git?

Github makes it super-easy to develop and share code. If you prefer to share code otherwise, you are free to do so.

Closed-source projects

If your code is sensitive and cannot be disclosed, you would require to purchase Commercial license for Agile Toolkit. To start - download Agile Toolkit as you normally would as you will be developing in closed environment. When your site goes Live and becomes accessible by your users, you would require to have a valid license.

How to get it?

Register with www.agiletoolkit.org, add your site and purchase license for your site. You can also purchase personal license which will be valid for all the sites you own.

Licensing Questions and Answers

I am Freelancer / Web development company. If we use ATK, will our client need to pay extra?

No. The Agile Toolkit licenses is issued to developers. If you have 2 developers in your company you will need two licenses. You can develop unlimited amount of software based on Agile Toolkit and sell / distribute it to clients with no licensing restrictions. If your clients are only using software you made for them â€” they do not need to purchase license.

If you don't normally use Agile Toolkit, you can also purchase license for single domain, and pass those cost to your client. Single-domain license never expires and is one-time payment.

We own proprietary Agile Toolkit software built by a 3rd party. Can we modify it?

You would need a licensed developer to do modifications. You can find some companies on Engage page.

How can we switch between free and proprietary licenses?

Open-source and Commercial distributions are using same code. You must only register your site on www.agiletoolkit.org.
