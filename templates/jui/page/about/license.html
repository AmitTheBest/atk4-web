Dual Licensing of Agile Toolkit

Agile Toolkit uses a Dual License business model for its software development modules and offers licenses for commercial and opens source development.

Using Open-source License

Our Open-source license is most suitable for other open-source projects as well as your internal projects.

Open-Source projects

We believe in open-source software. Our decision not to provide Agile Toolkit under MIT license is in order to support Open-Source movement. If you are using a free version of Agile Toolkit it comes with AGPL license.

Many web applicaitons such as Drupal, Wordpres, phpMyAdmin and more are using GPL license. Unfortunatelly a regular GPL license does not enforce availability of source code if someone builds a commercial web service using GPL code. Affero GPL license addresses that by requesting that a download source link would be present.

Commercial projects

In many cases publishing your software source code is more beneficial even in commercial project. For example, you can take an free open-source AGPL web project, build a hosted service with subscription then charge users for your hosted services. They still can download your software and use it locally. This is a great opportunity for you to promote your on-line resource. Think wordpress.com or kaltura.com.

If you are building highly customised software for your own use, still — open-source it. It will help others to learn the toolkit or perhaps even help you to improve your softawre

Using Proprietary License

Ourselves and other commercial companies develop web projects using Agile Toolkit on a daily basis. Those companies are using commercial license of the Agile Toolkit. Apart from the license agreement, nothing in the source code is different.

Proprietary projects with sensetive Intellectual Property

Typically if you are building a start-up or getting investors for your web venture, they would want you to keep your intellectual property hidden. You would need proprietary license if you wish to work on such projects.

Enterprise projects

In larger organisations it is much more beneficial to use commercial verison of Agile Toolkit. This ensures that you are receiving proper commercial support.

Freelancers

If you are a freelancer, take your time to learn Agile Toolkit using open-source version. However we are pretty sure that by using Agile Toolkit in your projects, you will be able to reduce cost by much more than a monthly license fee of Agile Toolkit.

Offshore development companies

This is a great opportunity for you to list your services on your Jobs page. You can list yourself even if you are not licensed, however you are much more likely to win projects if you are holding active license.

I am Freelancer / Web development company. If we use ATK, will our client need to pay extra?

No. Agile Toolkit licenses are issued to developers. If you have 2 developers in your company you will need two licenses. You can develop unlimited amount of software based on Agile Toolkit and sell / distribute it to clients with no licensing restrictions. If your clients are only using software you made for them — they do not need to purchase license.

After software is deployed, will we need to continue pay for licensing?

Only if you are developing it. If you require only occasinal changes, ask some licensed developer to do them for you.

We own proprietary software built by 3rd party. Can we modify it?

You would need a licensed developer to do modifications. This is to ensure that your software quality do not degrade. However you can freely change templates if you only need to do visual changes.

Alternatively you can open-source your software. In this case you can alwasy ask support from community if you encounter problem.

How can we switch between free and proprietary licenses?

If you have purchased proprietary license, you will have access information for download of an up-to-date commercial ATK distribution. Simply replace your "atk4" folder.

You can always download open-source ATK from download page. For how long commercial license is purchased?

Commercial license must be renewed annualy. If for some reason you no longer develop uning Agile Toolkit, you do not need to renew your license. All your existing software project can continue to run.
