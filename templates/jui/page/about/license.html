Dual Licensing of Agile Toolkit

Agile Toolkit offers licenses for commercial and open source development.

Open-source License

Our open-source license is most suitable for other open-source projects as well as your internal projects.

Open-Source projects

We believe in open-source software so we have made the open-source version available free of charge. If you are using an open-source version of Agile Toolkit it comes with AGPL license.

Many web applications such as Drupal, Wordpres, phpMyAdmin and more are using GPL license. Unfortunately a regular GPL license does not enforce availability of source code if someone builds a commercial web service using GPL code. Affero GPL license addresses that by requesting that a download source link would be present.

Commercial projects

In many cases publishing your software source code is more beneficial even in commercial project. For example, you can take an free open-source AGPL web project, build a hosted service with subscription then charge users for your hosted services. They still can download your software and use it locally. This is a great opportunity for you to promote your on-line resource. Think wordpress.com or kaltura.com.

If you are building highly customised software for your own use, still — open-source it. It will help others to learn the toolkit or perhaps even help you to improve your software

Using Commercial License

100s of commercial web projects have been developed using the Agile Toolkit under the commercial license. This ensures that the source code is fully owned by your company.

Start ups

Typically if you are building a start-up or getting investors for your web venture, they would want you to keep your intellectual property hidden. You would need a commercial license if you wish to work on such projects.

Enterprises

For business critical software, the commercial license ensures that you receive full support for our development issues.

Freelancers

If you are a freelancer, you can learn how to use the Agile Toolkit using open-source software. Once you are working with clients, we would advise you to upgrade to the commercial license so that the code that you write for them is covered by the commercial terms. You can also list your services on the Jobs page.

Offshore development companies

This is a great opportunity for you to list your services on your Jobs page. You can list yourself even if you are not licensed, however you are much more likely to win projects if you are holding active license.

I am Freelancer / Web development company. If we use ATK, will our client need to pay extra?

No. The Agile Toolkit licenses is issued to developers. If you have 2 developers in your company you will need two licenses. You can develop unlimited amount of software based on Agile Toolkit and sell / distribute it to clients with no licensing restrictions. If your clients are only using software you made for them — they do not need to purchase license.

After software is deployed, will we need to continue pay for licensing?

Only if you are developing it. If you require only occasional changes, you could ask a licensed developer to do them for you.

We own proprietary software built by a 3rd party. Can we modify it?

You would need a licensed developer to do modifications. This is to ensure that your software quality do not degrade. However you can freely change templates if you only need to do visual changes.

Alternatively you can open-source your software. In this case you can always ask support from community if you encounter problem.

How can we switch between free and proprietary licenses?

If you have purchased a commercial license, you can download the up-to-date commercial ATK distribution license. Simply replace your "atk4" folder.

You can download the open-source ATK from the download page. For how long is the commercial license purchased?

The commercial license must be renewed annually. The license is for development work only. If for some reason you stop developing using the Agile Toolkit, there is no need to renew your license. All your existing software project developed in the Agile Toolkit can continue run regardless of whether you have a license or not.
