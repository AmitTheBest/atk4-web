Knowledge Requirements

Agile Toolkit is used to power many high end projects and it does not lack in flexibility or power. Nevertheless, developing with Agile Toolkit is much simpler than developing with plain PHP alone. Once you start using Agile Toolkit, it's unlikely that you'll want to mess with things on the low level of PHP again. It makes a great deal of sense to skip some low-level stuff and get real results quick.

It turns out that you need to know very little to use Agile Toolkit.

Programming in PHP

It might sound sad, but PHP is not the best language with which to learn programming. You can write code easily, but seeing the results and experimenting with code can be quite challenging - much more complicated than QBASIC used to be.

Nevertheless, you do need to know about PHP expressions, variables, control structures and logical operators. So if you don't, start reading the PHP language reference. You might want to skip the features which Agile Toolkit specifically does not use (as noted on the previous page)

Object-Oriented Programming Training

Apart from the syntax, you need to understand how objects work. Look for some on-line documentation on Object-Oriented Programming for more information. You will need to understand the difference between a Class and an Object, how class inheritance works, and how you can override methods. Some experience in object-oriented development is really beneficial when using Agile Toolkit, even if you used a language other than PHP.

Understanding Performance

One of the reasons why Relational Databases are used is ability to traverse relations within regular queries. Agile Toolkit 4.2 creates an amazing interface to joins, sub-selects and expressions. If you are using relational database, there are no excuse to iterate through results of a query only to execute additional query in the loop. A well written project will execute no more than 10 queries for the rendering of a single page. Refer to Agile Toolkit performance guide for more information.
