Why Object Oriented Web Programming is more powerful than conventional MVC?

OOP is one of the major concepts in the world of Computing. For decades Object Oriented Programming has been used in all environments - on computer desktop, on mobile devices. Some web frameworks have been using it.

You might say now - "I'm using MVC, so I am using OOP!". The truth is, even though most of the PHP Web Frameworks claim to be Object-Oriented, that does not imply that the developers using them are producing a well thought-out Object Oriented Code.

Agile Toolkit implements a solid Object-Oriented core framework, which you extend in your software. It might require you to learn some new ways, but it's a step into the right direction. If you have recently learned some of the frameworks such as Ruby on Rails, the Agile Toolkit will be a significant step forward. After all — Agile Toolkit takes web development to a whole new level with it's integrated User Interface.

What is Wrong With Existing MVC Frameworks?

Similarly to the development of static web sites, most MVC frameworks are page-oriented. You've got one controller per page and most likely one model/template per page. They wouldn't attempt to abstract any of the technologies they rely on such as HTML, JavaScript, Templates, jQuery and they make no efforts to be flexible when it comes to add-ons or re-use of your own code across multiple projects.

Those limitations are the consequences of their decisions to use very simple Object-Oriented. For example, Views are often PHP files containing HTML or Logic-infused templates which can't be extended, Models could be technically extended, but there are no benefit in doing that.

Your product would most likely have one controller per page, one model per SQL table and a LOT of HTML code duplication in your views.

Need More info about Object-Oriented Programming?

Objects are part of our real life. Take a car, for example. A car is an object. It has properties, such as color, model, year. There are things you can do with a car: drive, wash, lock and unlock it.

The goal of Object Oriented Programming is to discover self-sufficient objects in the software, and describe them using classes. The behaviour and the properties of the button are described in the Button class.

Properly constructed Object-oriented architecture hides the complexity of things, while still allowing a developer to be in control of the behaviour. With the great Object-oriented capabilities introduced to PHP since PHP5, web applications based on PHP can also be very object-oriented.

Agile Toolkit implements a general object-oriented architecture, as well as large amount of base classes. You can use the classes you like, and build upon them. Programming with Agile Toolkit is like building a sky-scraper: you build upon the foundation. You re-use, extend, enhance the basic building blocks.

Agile Toolkit Core Concepts

The "View" class

$page->add('Button')->set('This is a "Button" View');

The first fundamental concept of Agile Toolkit are the "View" classes. If you have been developing on GUI libraries, you know what it is.

The button you see on the right is instance of the class "Button". It takes just one line of code to add a button anywhere on your page. The Button object then takes care of rendering itself, handling events, using jQuery UI's widget. This ability of object to do what's necessary is called "abstraction".

Agile Toolkit has LOTS of view classes for you to use, but if you need to have your own class, you can easily create it. You can override template for any view to specify your own HTML-template, so adding your custom pieces of HTML code is as easy as typing it's filename.

The "Render Tree"

$frame = $page->add('HtmlElement')->addStyle('border','1px dashed red')->addStyle('padding','5px'); $frame = $frame->add('HtmlElement')->addStyle('border','1px dashed green')->addStyle('padding','5px'); $frame->add('Button')->set('Same Button Class');

Inside your application you add objects into other objects. Above you see that the exactly same button is not added into the "HtmlElement" class with a custom border, which is placed inside yet another HtmlElement. The recursive rendering is done automatically by Agile Toolkit, so your job as developer is to put projects into the right places and configure them to use right styles and have proper labels.

The "Model" Class

$page->add('FormAndSave')->setModel('Person')->load($person_id);

The next fundamental concept is the Model Class. Model describes your data entity including fields, labels, validation rules, custom actions, hints and more. With an easy syntax you can associate "Model" object with the "View" object. This connections makes use of an appropriate "Controller" which job is to "configure" the view for the Model.

You can mix and match. FormAndSave view is a descendant of a "Form" class which automatically adds an "Update" button and allows you to save data into the database.

The "Controller" Class

$model->add('Controller_Encrypt')->useField('account_number')->useEncryption('rsa'); $api->add('Controller_PatternRouter') ->addRule("profile/(.*)", "profile/details", array("profile_id")) ->addRule("blog/([0-9]*)-.*", "blog/item", array("id")) ->route();

Controllers in Agile Toolkit are classes which "Enrich" the functionality of existing objects. The "Controller_Encrypt" adds ability into a "Model" to use strong encryption on some of the fields. The PatternRouter controller adds pattern matching and PHP-based rewrites.

Controllers are used when you connect to database, associate view with model, use pagination on the form, add "strength" indicator into the password field or add anti-XSF tokens into the form class.

The "Application" Class

Application class is designed so that you can create Your Own Toolkit. In this new application you might pre-initialize the controllers you want and include the add-ons you would most probably use. Finally you will be able to throw your snippets away and turn them into on-demand classes.

The "Agile Store"

Given the powerful architecture of Agile Toolkit, many different extensions, templates, models and the whole UI pages. The "Agile Store" which is introduced in 4.2 of Agile Toolkit is the way of us saying - "Got a nice piece of code? Share and perhaps even make some money on it!". Or if you are looking for a good quality add-on, why purchase it from someone for the price of a "Frappuchino".

Some examples of what you can find in the store are: "Comment Anywhere" - plug-in you can put on any page which enables users to comment on anything, "ModelAudit" - controller to make your models auditable (records editing operations into separate table) and also brings the views for you to look at the audit history. "Countries" - delivers model containing up-to-date list of countries. "Job Queue" - a system for implementing your own Cloud-Powered Job Processing queues based on SQL or SQS. Many others too.
