What exactly is Agile Toolkit? Is it a framework?

Agile Toolkit is a development environment built on top of the PHP5 language. However, developing with Agile Toolkit is easier than using pure PHP5. The Agile Toolkit has it's own rules, concepts, policies and best practices. In documentation and everywhere else we refer to Agile Toolkit as simply "toolkit" or "framework".

If you are familiar with JavaScript frameworks, the Agile Toolkit is similar in concept and design to jQuery.

Is it a new framework?

Agile Toolkit was known previously as AModules and traces it's history to 1999. Each major release have been a major rewrite, although some of the code was just cleaned up and ported. Throughout it's history the toolkit have powered over 300 commercial projects. In 2010 it was decided to release Agile Toolkit publicly under open-source and commercial licenses

Agile Toolkit uses many of the concepts found in other frameworks, although they are implemented independently, often differently. Some features pre-dates appearance of their analogues in other frameworks. See how Agile Toolkit has evolved on Agile Toolkit Timeline

Why is it called Agile?

Originally it is named after the company who published it — Agile Technologies. We are considering picking a different name, but until we get a really good one we are reluctant to change it. Suggest a name.

I am beginner, will I understand it?

Yes. The core concept of Agile Toolkit is really simple. It breaks down web page or web software into blocks. As a beginner you can use existing blocks to build your software. You do not need to know HTML or CSS in order to get some serious results.

We have made available a learning documents which will guide you through a basic concepts of the toolkit. However, you will need your own Web server and PHP in order to install toolkit and experiment with it

I am hardcore developer / hacker. Will it restrict me?

No. Agile Toolkit is based on several powerful concepts which can be further extended and personalized. This makes Agile Toolkit viable for a extremely big and complex projects, where other frameworks usually start either being too slow, or too stiff

As a power user you will find it much easier to build and deploy commercial software.

Why another PHP framework?

Being involved in commercial projects for over 10 years we understand the short-comings of PHP as a platform when it comes to professional work. When a new company looks to build a world-changing web application, they go through steps of fund-raising and prototype-building. When a company needs to create a good looking web software within 1-month time, most of the problems frameworks are trying to solve fade away in front of the major requirement — efficiency. None of the frameworks out there are sufficiently efficient in terms of development time, execution speed and code re-use.

Agile Toolkit has a tremendous success in Software as a Service web applications as well as web administrative tools due it's efficiency and flexibility, however we are also aiming to make things easier for the open-source community.

Despite many of the PHP frameworks being free, they are using Free MIT license which does not require developers to open their projects or pay. We believe that if you get something for free, you should give something in return. For that reason Agile Toolkit is using AGPL open-source license. We would like to see many high-quality open-source web projects spawn around Agile Toolkit, exchanging extensions, skins and source code.

We ask you to support open-source movement by using AGPL license for your software, extension or theme. At the same time, make your software available commercially through the enterprise license.

How Agile Toolkit approach compares with other UI frameworks?

We have found that other frameworks often have one of the following 3 disadvantages:

  • Too difficult to use. Simple stuff takes lots of code. Vaadin, Zend
  • Distant from Web Experience. Not sufficiently flexible. Mocha, Cappuchino
  • Lacking JavaScript extensibility. Using their own JavaScript APIs. KoolPHP, Sigma
