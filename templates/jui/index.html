Features

Web Software — reinvented

You might have thought that web development is about producing HTML code. Not anymore. Objects inside Agile Toolkit are alive. They automatically interact with data, other objects, JavaScript and render themselves when necessary. It's a new, best way for robust web software.

Re-use of your code

You might have thought that web development is about producing HTML code. Not anymore. Objects inside Agile Toolkit are alive. They interact with models, data, other objects, JavaScript, and render themselves when necessary. It’s a new, best way for robust web software.

Keep simple things simple

If you compare the basic components in different frameworks, the classes in Agile Toolkit are much cleaner and more basic. However they are extremely extensible and can gain new features, look, bindings, type support, data bindings through simple actions by developers.

Best choice for your business

Agile Toolkit is designed with your business needs in mind. It is easy to learn and suitable for large projects. You can build prototypes faster and integrate legacy applications with the web. Many business-critical extensions are included such as documentation, testing suites and more.

Project Growth

We all get worried that our code might turn into a spaghetti after the business team changes their mind for 3rd time! Agile Toolkit provides a great architecture for keeping code organised but it also enables you to perform code reorganisation much more efficiently.

Simple templates

Agile toolkit comes with integrated template engine which is simple and fast. This is a template your designers can easily understand and involves no programming. The same template also allows you to preserve the look of your mock-ups even after they are changed into templates.

Best choice for open-source

If you plan to develop an open-source project, Agile Toolkit will provide you with dynamic and flexible platform which you and your contributors will easily understand. Your project will contain clean, object-oriented code ensuring future and growth of your project.

Rich Web Applications

The unique approach of Agile Toolkit means that you can write business code on the server side which maintains control over the user interface. As a result, the interface interacts with the backend through AJAX in a very efficient way which is really easy to keep track of.

"Real" Plugin Support

The concept of an add-on bundle is much more powerful in Agile Toolkit. Add-ons include interface elements and models, which you can use inside your own UI. They are easy to integrate, extend, customise and when you upgrade an add-on, you don't lose your modifications.
